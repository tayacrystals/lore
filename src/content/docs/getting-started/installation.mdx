## Prerequisites

Before you begin, make sure you have the following installed:

- **Node.js** version 18 or higher
- **bun**, **npm**, or **pnpm** as your package manager

<Callout type="tip">
  We recommend using [bun](https://bun.sh) for the fastest installation experience.
</Callout>

## Setup

<Steps>
  <Step title="Create a new project">
    Start by creating a new Astro project using the minimal template:

    ```bash
    bun create astro@latest my-docs -- --template minimal
    ```
  </Step>

  <Step title="Install dependencies">
    Install all required packages:

    <Tabs id="pkg-manager">
      <Tab label="bun">
        ```bash
        bun add @astrojs/mdx astro-expressive-code tailwindcss @tailwindcss/vite
        ```
      </Tab>
      <Tab label="npm">
        ```bash
        npm install @astrojs/mdx astro-expressive-code tailwindcss @tailwindcss/vite
        ```
      </Tab>
      <Tab label="pnpm">
        ```bash
        pnpm add @astrojs/mdx astro-expressive-code tailwindcss @tailwindcss/vite
        ```
      </Tab>
    </Tabs>
  </Step>

  <Step title="Configure Astro">
    Update your `astro.config.mjs` to include the required integrations:

    ```js
    // astro.config.mjs
    import { defineConfig } from "astro/config";
    import tailwindcss from "@tailwindcss/vite";
    import expressiveCode from "astro-expressive-code";
    import mdx from "@astrojs/mdx";

    export default defineConfig({
      integrations: [expressiveCode(), mdx()],
      vite: {
        plugins: [tailwindcss()],
      },
    });
    ```
  </Step>

  <Step title="Start the dev server">
    Run the development server to see your site:

    ```bash
    bun run dev
    ```

    Open [http://localhost:4321](http://localhost:4321) in your browser.
  </Step>
</Steps>

<Callout type="note">
  Make sure `expressiveCode()` comes **before** `mdx()` in the integrations array. This is required for syntax highlighting to work correctly in MDX files.
</Callout>

## Project Structure

After setup, your project should look like this:

```
my-docs/
├── src/
│   ├── content/docs/     # Your documentation pages
│   ├── components/       # Reusable components
│   ├── layouts/          # Page layouts
│   ├── pages/            # Route pages
│   └── styles/           # Global styles
├── public/               # Static assets
├── astro.config.mjs      # Astro configuration
└── package.json
```
