## Introduction

This guide will teach you how to write effective documentation using MDX. MDX combines Markdown with JSX components, allowing you to create rich, interactive documentation.

<Callout type="note">
  MDX allows you to use both Markdown syntax and custom components in the same file.
</Callout>

## Getting Started with MDX

MDX files use the `.mdx` extension and support all standard Markdown features plus custom components.

### Basic Markdown Syntax

You can use all standard Markdown features:

- **Bold text** with `**text**`
- *Italic text* with `*text*`
- `Inline code` with backticks
- [Links](https://example.com) with `[text](url)`

### Code Blocks

Code blocks support syntax highlighting:

```javascript
// JavaScript example
function greet(name) {
  return `Hello, ${name}!`;
}

console.log(greet('World'));
```

```typescript
// TypeScript example
interface User {
  name: string;
  role: string;
}

const user: User = {
  name: 'Alice',
  role: 'Developer'
};
```

## Blockquotes

> This is a blockquote. Use it for quotes, important notes, or to highlight specific content.
>
> You can have multiple paragraphs in a blockquote.

## Best Practices

<Callout type="tip">
  Keep your documentation organized with clear headings and logical structure. Use components wisely to enhance readability.
</Callout>

When writing documentation:

- Use clear, concise headings
- Break up long sections with subheadings
- Include code examples where helpful
- Use callouts to highlight key information
- Structure step-by-step instructions clearly
