## Integration Config

The `lore()` integration accepts the following options:

```ts
lore({
  // Site title shown in sidebar header and page titles
  title: "My Docs",

  // Primary accent hue (0–360 on OKLCH color wheel)
  primaryHue: 55,

  // Sidebar group configuration
  sidebar: {
    "getting-started": { label: "Getting Started", order: 1 },
    guides: { label: "Guides", order: 2 },
    api: { label: "API Reference", order: 3 },
  },

  // Social links shown in sidebar footer
  social: {
    github: "https://github.com/your-org/your-repo",
  },

  // Additional CSS files to load
  customCss: ["./src/styles/custom.css"],
})
```

### Options Reference

| Option       | Type                                                   | Default | Description                                 |
| ------------ | ------------------------------------------------------ | ------- | ------------------------------------------- |
| `title`      | `string`                                               | `"Docs"` | Site title                                 |
| `primaryHue` | `number`                                               | `55`    | Accent color hue (0–360, OKLCH color wheel) |
| `sidebar`    | `Record<string, SidebarGroup>`                         | `{}`    | Sidebar group labels, ordering, and nesting |
| `social`     | `{ github?: string }`                                  | `{}`    | Social links in sidebar footer              |
| `customCss`  | `string[]`                                             | `[]`    | Additional CSS files to import              |

## Primary Hue

The `primaryHue` option controls the accent color used throughout the site — links, active states, selections, and focus rings. It maps to an OKLCH hue value (0–360):

```ts
lore({
  primaryHue: 55,   // warm amber (default)
})
```

Some common hue values:

| Hue   | Color        |
| ----- | ------------ |
| `0`   | Pink/Red     |
| `30`  | Orange       |
| `55`  | Amber        |
| `90`  | Yellow-Green |
| `145` | Green        |
| `210` | Blue         |
| `270` | Purple       |
| `330` | Magenta      |

<Callout type="tip">
  The hue is applied via the `--fd-hue` CSS custom property. All theme colors derived from it update automatically in both light and dark mode.
</Callout>

## Sidebar Groups

Sidebar groups are defined in the `sidebar` config option. Each key is a folder name under `src/content/docs/`, and the value sets its label and sort order.

```ts
sidebar: {
  "getting-started": { label: "Getting Started", order: 1 },
  guides: { label: "Guides", order: 2 },
}
```

### Nested Groups

Groups can contain `children` for nested sidebar sections. Each child key corresponds to a subfolder:

```ts
sidebar: {
  guides: {
    label: "Guides",
    order: 1,
    children: {
      advanced: { label: "Advanced", order: 1 },
    },
  },
}
```

<Callout type="tip">
  Create a new folder in `src/content/docs/` and add it to the `sidebar` config to create a new section.
</Callout>

## Frontmatter Schema

Each documentation page supports these frontmatter fields:

| Field         | Type      | Default | Description                     |
| ------------- | --------- | ------- | ------------------------------- |
| `title`       | `string`  | ---     | Page title (required)           |
| `description` | `string`  | ---     | Page description for SEO        |
| `order`       | `number`  | `999`   | Sort order within sidebar group |
| `group`       | `string`  | ---     | Override sidebar group          |
| `icon`        | `string`  | ---     | Icon for the page               |
| `toc`         | `boolean` | `true`  | Show table of contents          |
| `draft`       | `boolean` | `false` | Hide from production build      |

<Accordion title="How does ordering work?">
  Pages are sorted by their `order` field within each sidebar group. Lower numbers appear first. If two pages have the same order, they're sorted alphabetically by title.

  Groups are also sorted by their `order` field in the `sidebar` configuration.
</Accordion>

## Custom CSS

Use the `customCss` option to load additional stylesheets:

```ts
lore({
  title: "My Docs",
  customCss: ["./src/styles/custom.css"],
})
```

Your custom CSS can override any of the built-in design tokens. See [Styling](/docs/customization/styling) for details.

## Dark Mode

Dark mode is handled automatically with:

1. **System detection** --- respects `prefers-color-scheme`
2. **Manual toggle** --- persisted to `localStorage`
3. **No flash** --- inline script in `<head>` prevents FOUC

The dark theme overrides are defined using the `.dark` selector in the integration's CSS.
